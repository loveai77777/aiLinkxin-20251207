# 网站启动说明

## 问题诊断

如果网站无法访问 `http://localhost:3000`，通常是因为：

1. **依赖未安装** - `node_modules` 目录不存在
2. **Node.js 未安装** - 系统找不到 `node` 命令
3. **服务器未启动** - 开发服务器没有运行

## 解决步骤

### 方法 1：使用批处理文件（最简单）

1. **双击运行** `启动网站.bat` 文件
2. 脚本会自动：
   - 检查 Node.js 是否安装
   - 安装依赖（如果需要）
   - 启动开发服务器

### 方法 2：手动在终端运行

#### 步骤 1：打开终端

- **方法 A**：在项目文件夹中，按住 `Shift` + 右键点击空白处，选择"在此处打开 PowerShell 窗口"
- **方法 B**：按 `Win + R`，输入 `powershell`，然后 `cd` 到项目目录

#### 步骤 2：进入项目目录

```powershell
cd "c:\Users\mirag\Desktop\my project\xinyuliu-main\main-aili8nkxin-1130"
```

#### 步骤 3：检查 Node.js

```powershell
node --version
npm --version
```

如果显示"找不到命令"，需要先安装 Node.js：
- 访问 https://nodejs.org/
- 下载并安装 LTS 版本
- 安装后重启终端

#### 步骤 4：安装依赖

```powershell
npm install
```

这个过程可能需要 2-5 分钟，请耐心等待。

#### 步骤 5：启动服务器

```powershell
npm run dev
```

看到类似以下输出表示成功：
```
  ▲ Next.js 14.2.0
  - Local:        http://localhost:3000
```

#### 步骤 6：打开浏览器

访问：**http://localhost:3000**

## 常见问题

### Q: 提示 "npm 不是内部或外部命令"

**A:** Node.js 未安装或未添加到 PATH。请：
1. 安装 Node.js（从 https://nodejs.org/）
2. 安装时选择"添加到 PATH"选项
3. 重启电脑或重新打开终端

### Q: npm install 很慢或失败

**A:** 可能是网络问题，尝试：
```powershell
npm install --registry https://registry.npmmirror.com
```

### Q: 端口 3000 已被占用

**A:** 可以指定其他端口：
```powershell
npm run dev -- -p 3001
```
然后访问 http://localhost:3001

### Q: 看到编译错误

**A:** 检查：
1. Node.js 版本是否 >= 18
2. 所有依赖是否安装成功
3. 查看具体错误信息并修复

## 验证安装

安装完成后，应该能看到：
- ✅ `node_modules` 文件夹存在
- ✅ `package-lock.json` 文件已更新
- ✅ 运行 `npm run dev` 时没有错误

1. Node.js 版本（运行 `node --version`）
2. npm 版本（运行 `npm --version`）
3. 具体的错误信息
