## 🚀 方案 1: 使用 Vercel（推荐 - 最简单）

### 为什么选择 Vercel？
- ✅ **完全自动**：连接 GitHub 后，推送代码自动部署
- ✅ **专为 Next.js 优化**：Next.js 官方推荐的部署平台
- ✅ **完全免费**：个人项目免费使用
- ✅ **全球 CDN**：网站加载速度快
- ✅ **自动 HTTPS**：免费 SSL 证书

### 设置步骤（只需 5 分钟）

#### 步骤 1: 注册 Vercel
1. 访问 https://vercel.com
2. 点击 **"Sign Up"**
3. 选择 **"Continue with GitHub"**（使用你的 GitHub 账号登录）

#### 步骤 2: 导入项目
1. 登录后，点击 **"Add New..."** → **"Project"**
2. 在 "Import Git Repository" 中，找到并选择你的仓库：**`loveai77777/aiLinkxin-20251207`**
3. 点击 **"Import"**

#### 步骤 3: 配置项目（使用默认设置即可）
- ✅ Framework Preset: **Next.js**（自动检测）
- ✅ Root Directory: `./`（默认）
- ✅ Build Command: `npm run build`（默认）
- ✅ Output Directory: `.next`（默认）
- ✅ Install Command: `npm install`（默认）

#### 步骤 4: 部署
1. 点击 **"Deploy"** 按钮
2. 等待 1-3 分钟，构建完成
3. 部署成功！你会看到一个临时域名，例如：`aiLinkxin-20251207.vercel.app`

#### 步骤 5: ✅ 自动部署已启用！

**完成！** 现在：
- ✅ Vercel 已经连接到你的 GitHub 仓库
- ✅ 每次你推送到 `main` 分支，Vercel 会：
  - 自动检测到代码更改
  - 自动重新构建网站
  - 自动部署新版本
  - 网站会在 1-3 分钟内更新

## 📝 日常更新流程（超级简单）

以后更新网站只需要：

1. **在本地修改代码**
   - 使用你喜欢的编辑器修改文件

2. **提交并推送到 GitHub**
   ```bash
   git add .
   git commit -m "更新网站内容"
   git push
   ```
   
   或者使用 GitHub Desktop：
   - 填写提交信息
   - 点击 "Commit to main"
   - 点击 "Push origin"

3. **自动部署**
   - Vercel 会自动检测到推送（几秒钟内）
   - 自动开始构建（1-3 分钟）
   - 网站自动更新到最新版本

**就这么简单！** 🎉

## 🔍 如何查看部署状态

### 在 Vercel 上查看
1. 登录 https://vercel.com
2. 点击你的项目
3. 在 "Deployments" 标签页可以看到所有部署记录
4. 绿色 ✅ 表示部署成功，红色 ❌ 表示失败

### 在 GitHub 上查看
1. 访问你的仓库：https://github.com/loveai77777/aiLinkxin-20251207
2. 点击 "Actions" 标签页（如果使用 GitHub Actions）
3. 可以看到部署工作流的状态

## 🎯 绑定自定义域名（可选）

如果你有自己的域名（如 www.ailinkxin.com）：

1. 在 Vercel 项目页面，点击 **"Settings"** → **"Domains"**
2. 输入你的域名：`www.ailinkxin.com`
3. 按照提示在你的域名注册商（GoDaddy、Namecheap 等）配置 DNS：
   - **类型**: CNAME
   - **名称**: www
   - **值**: `cname.vercel-dns.com`
4. 等待 DNS 生效（几分钟到几小时）
5. Vercel 会自动配置 HTTPS 证书

## 🔧 方案 2: 使用 GitHub Actions + Vercel（更灵活）

如果你想要更多的控制，可以配置 GitHub Actions 来自动触发 Vercel 部署。

### 优点
- 可以添加部署前检查
- 可以运行测试
- 可以自定义部署流程

### 设置
1. 在 Vercel 项目设置中，找到 "Git" 部分
2. 确保 "Automatic deployments from Git" 已启用（默认已启用）

## ❓ 常见问题

### Q: 推送代码后网站没有自动更新？
**A:** 检查以下几点：
1. 确认代码已成功推送到 GitHub 的 `main` 分支
2. 登录 Vercel，查看 "Deployments" 是否有新的部署记录
3. 如果部署失败，查看错误日志并修复问题

### Q: 构建失败怎么办？
**A:** 
1. 在 Vercel 项目页面，点击失败的部署记录
2. 查看构建日志，找到错误原因
3. 修复代码中的错误后重新推送

### Q: 如何回退到之前的版本？
**A:**
1. 在 Vercel 项目页面的 "Deployments" 中
2. 找到之前的成功部署
3. 点击 "..." → "Promote to Production"

### Q: 可以手动触发部署吗？
**A:** 可以！
1. 在 Vercel 项目页面，点击 "Deployments"
2. 点击 "Redeploy" 按钮
3. 或者重新推送一次代码
