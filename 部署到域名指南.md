## 🚀 完整部署步骤

### 第一步：在 Vercel 上部署（如果没有部署）

#### 1.1 访问 Vercel 并登录
1. 访问 **https://vercel.com**
2. 点击 **"Sign Up"** 或 **"Log In"**
3. 选择 **"Continue with GitHub"**（使用你的 GitHub 账号登录）

#### 1.2 导入项目
1. 登录后，点击右上角的 **"Add New..."** → **"Project"**
2. 在 "Import Git Repository" 中找到你的仓库：**`loveai77777/aiLinkxin-20251207`**
3. 如果看不到，点击 **"Adjust GitHub App Permissions"** 授权访问
4. 选择仓库后，点击 **"Import"**

#### 1.3 配置项目（通常使用默认设置即可）
- ✅ **Project Name**: `aiLinkxin-20251207`（或自定义为 `ailinkxin-website`）
- ✅ **Framework Preset**: `Next.js`（自动检测）
- ✅ **Root Directory**: `./`（默认）
- ✅ **Build Command**: `npm run build`（默认）
- ✅ **Output Directory**: `.next`（默认）
- ✅ **Install Command**: `npm install`（默认）

#### 1.4 环境变量（如果需要）
如果项目使用了环境变量（如 `DIFY_API_KEY`），在 "Environment Variables" 部分添加：
- `DIFY_API_BASE_URL`: 你的 Dify API 地址
- `DIFY_API_KEY`: 你的 Dify API 密钥

#### 1.5 开始部署
1. 点击 **"Deploy"** 按钮
2. 等待 1-3 分钟，构建完成
3. 你会看到一个临时域名，例如：`aiLinkxin-20251207.vercel.app`

---

### 第二步：绑定自定义域名 www.ailinkxin.com

#### 2.1 在 Vercel 中添加域名
1. 在 Vercel 项目页面，点击顶部的 **"Settings"** 标签
2. 在左侧菜单中找到 **"Domains"**
3. 在 "Domains" 输入框中输入：`www.ailinkxin.com`
4. 点击 **"Add"** 按钮

#### 2.2 查看 DNS 配置要求
添加域名后，Vercel 会显示需要配置的 DNS 记录，通常是：

**方法 1：使用 CNAME 记录（推荐）**
- **类型**: `CNAME`
- **名称**: `www`
- **值**: `cname.vercel-dns.com`
- **TTL**: `3600`（或使用默认值）

**方法 2：使用 A 记录（如果 CNAME 不支持）**
- Vercel 可能会显示多个 A 记录，按提示添加

---

### 第三步：在域名注册商配置 DNS

#### 3.1 登录你的域名注册商
根据你的域名注册商（常见的有）：
- **GoDaddy**: https://www.godaddy.com
- **Namecheap**: https://www.namecheap.com
- **阿里云**: https://www.aliyun.com
- **腾讯云**: https://cloud.tencent.com
- **Cloudflare**: https://www.cloudflare.com
- 其他注册商

#### 3.2 找到 DNS 管理页面
1. 登录后，找到域名管理页面
2. 点击你的域名 `ailinkxin.com`
3. 找到 **"DNS 管理"** 或 **"DNS 设置"** 或 **"Name Servers"**

#### 3.3 添加 DNS 记录
添加以下记录：

**CNAME 记录**（推荐）：
```
类型: CNAME
名称: www
值: cname.vercel-dns.com
TTL: 3600（或自动）
```

**如果还想支持根域名（ailinkxin.com）**，需要添加：
```
类型: A
名称: @
值: 76.76.21.21（Vercel 提供的 IP，请查看 Vercel 上的具体值）
TTL: 3600
```

#### 3.4 保存 DNS 设置
1. 点击 **"保存"** 或 **"Add Record"**
2. 等待 DNS 记录生效

---

### 第四步：等待 DNS 生效并验证

#### 4.1 DNS 传播时间
- **通常**: 几分钟到 1 小时
- **最长**: 最多 48 小时（但通常不会这么久）

#### 4.2 检查 DNS 是否生效
可以使用以下工具检查：
- **DNS Checker**: https://www.whatsmydns.net/#CNAME/www.ailinkxin.com
- **在线工具**: 在 Google 搜索 "DNS propagation checker"

#### 4.3 在 Vercel 中验证
1. 回到 Vercel 项目页面的 **"Settings"** → **"Domains"**
2. 如果域名旁边显示 **✅ Valid Configuration**，说明 DNS 配置正确
3. 如果显示警告，请检查 DNS 记录是否正确

#### 4.4 访问网站
等待 DNS 生效后，访问：
- **https://www.ailinkxin.com**

Vercel 会自动配置 HTTPS 证书，无需手动操作！

## ❓ 常见问题

### Q: DNS 已经配置但 Vercel 还是显示无效？
**A:** 
1. 等待几分钟，DNS 传播需要时间
2. 清除浏览器缓存
3. 检查 DNS 记录是否正确（名称、值、类型）
4. 确认域名已正确添加到 Vercel

### Q: 想要同时支持根域名（ailinkxin.com）和 www？
**A:**
1. 在 Vercel 中添加两个域名：`ailinkxin.com` 和 `www.ailinkxin.com`
2. 配置 DNS：
   - `www`: CNAME → `cname.vercel-dns.com`
   - `@`: A 记录 → Vercel 提供的 IP 地址
3. Vercel 会自动将根域名重定向到 www（或反之，可在设置中配置）

### Q: 部署失败怎么办？
**A:**
1. 在 Vercel 项目页面，点击 "Deployments"
2. 查看失败的部署记录
3. 点击查看构建日志，找到错误原因
4. 修复错误后重新推送代码

### Q: 如何检查部署状态？
**A:**
- **Vercel**: 登录后，在项目页面的 "Deployments" 标签查看
- **GitHub**: 在仓库的 "Actions" 标签查看（如果使用了 GitHub Actions）
